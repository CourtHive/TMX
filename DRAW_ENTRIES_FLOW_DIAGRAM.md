## Data Structures

### Entry Data (after mapping)
```typescript
{
  participantId: string,
  participant: {
    participantName: string,
    participantType: string,
    person?: { ... },
    // ... participant details
  },
  entryStatus: string,           // e.g., 'DIRECT_ACCEPTANCE'
  entryStage: string,             // 'MAIN' or 'QUALIFYING'
  seedNumber?: number,            // 1, 2, 3, etc.
  ranking?: number,               // National/regional ranking
  ratings: {
    wtn?: number,                 // WTN rating value
    utr?: number,                 // UTR rating value
  },
  cityState?: string,             // 'New York, NY'
  flights: Array<{                // Draw assignments
    drawId: string,
    drawName: string,
    eventId: string,
  }>,
  searchText: string,             // Lowercase participant name for search
  status: string,                 // Mapped entry status display
}
```

## Key Integration Points

1. **Reuses existing seeding functions** from `src/components/tables/eventsTable/seeding/`:
   - `enableManualSeeding.ts`
   - `generateSeedValues.ts`
   - `clearSeeding.ts` (removeSeeding.ts)
   - `cancelManualSeeding.ts`
   - `saveSeeding.ts`

2. **Uses shared formatters** from `src/components/tables/common/formatters/`:
   - `participantFormatter.ts`
   - `ratingFormatter.ts`

3. **Uses shared sorters** from `src/components/tables/common/sorters/`:
   - `ratingSorter.ts`
   - `headerSortElement.ts`

4. **Uses shared editors** from `src/components/tables/common/editors/`:
   - `numericEditor.ts`

5. **Integrates with tournamentEngine**:
   - `getEvent({ eventId, drawId })`
   - `getParticipants({ ... })`
   - Seeding functions internally call engine methods

6. **Uses courthive-components**:
   - `cModal` for modal rendering
